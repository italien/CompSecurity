Injection SQL

Injection sql occur when untrusted data is sent to an interpreter as part of a command orquery.
The attacker's hostile data can trick the interpreter into executing unintended commands or accessing to sensitive data without authorization.
We choosed to implement this different leaks :

  • URL Based SQL Injection
  • Cookie-Based SQL Injection
  • SQL Injection Through HTTP Headers

Cross-Site Scripting (XSS)

XSS flaws appear when an application takes untrusted data and sends it to a web browserwithout any verification , proper validation or escaping.
The attackers allow to execute some scripts (like javascript script) in the user's browser which can hijack user sessions, redirect to malicious web sites or deface web sites.
